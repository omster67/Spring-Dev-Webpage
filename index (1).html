Description:

The purpose of this project is to teach students how 3D logic and Object Orientated Programming could be used to make complicated games online. I personally made this game to show my hobbies and interests and to also show how I like solving puzzles using algorithms.

Instructions:

Cube Controls:

  • U key: Top Face Rotation
  • D key: Bottom Face Rotation
  • L key: Left Face Rotation
  • R key: Right Face Rotation
  • F key: Front Face Rotation
  • B key: Back Face Rotation
  • J key: Horizontal Top Middle Face Rotation
  • N key: Horizontal Front Middle Face Rotation
  • M key: Vertical Front Middle Face Rotation
  • S key: Scramble

Learning Goal: Technical Analysis

1. Method with Parameters

Functionality: This method uses the k parameter to capture the user input from the keyboard. By pressing the key in the function, the program is able to determine what moves to be executed without using different functions to exceute every time.

2. Method with Return Value

Functionality: This method checks if the cubie is part of the rotation slice for the cube so it runs a coordinate check on the cubie and returns a boolean value. This is important because it tells the program what pieces to move or not.

LLM Assistance:

The LLM assisted in designing the HTML and CSS for the timer section of the cube and provided help for creating the base code for the cube. It also helped the way the M, N, and J middle slices work and the specific colors for each cubie. The LLM helped alot becase it added things that a normal game wouldn't have, like 3D logic, making the game very interactive.
